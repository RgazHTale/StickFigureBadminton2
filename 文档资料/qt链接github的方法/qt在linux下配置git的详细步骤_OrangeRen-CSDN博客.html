这里以Linux下码云gitee为例，将项目上传到github上于此差别不大
Windows下的配置点击这里：qt在window下配置git的详细步骤

步骤

  • 下载安装git，配置属性、生成SSH
  • 码云gitee添加git的SSH，创建仓库，复制仓库的SSH
  • 本地仓库建立，本地仓库链接远程仓库
  • qt creator中git的本地commit和远程push

下载安装git，配置属性、生成SSH

首先查看git是否安装git --version 如果没有安装就自行下载安装，同时注册一个gitee账号

下载完成后，配置属性
git config --global user.name xxx 用户名
git config --global user.email xxx.@xxx.com 自己注册的邮箱
ssh-keygen -t rsa -C xxx.@xxx.com 生成ssh密钥

根据提示的找到生成的ssh密钥文件(C:\Users\用户名\.ssh\id_rsa.pub /home/orgnge/.ssh/id_rsa.pub)，并将其复制到码云的ssh公钥中。

sudo gedit /home/orgnge/.ssh/id_rsa.pub

码云gitee添加git的SSH，创建仓库，复制仓库的SSH


在码云中创建一个仓库，并复制其仓库的SSH

本地仓库建立，本地仓库链接远程仓库

Linux用which git(Windows使用where git) 查看git的安装位置，并将其安装位置复制到qt creator的git设置中。

创建项目，选择添加到版本控制系统

将项目和gitee的新建的仓库链接(xxx改为之前远程仓库复制的SSH)

进入项目的路径，使用命令git remote add origin xxxx(也可以参考Windows下的链接方法)

之后进行commit和远程push，如果爆出警告和错误，就使用他的提示命令进行操作

本地仓库commit



远程仓库push

报出警告和错误



git push --set-upstream chatServer master
git push -u origin +master 这个是最后报上面更新被拒绝的解决办法

只要第一次提交完成之后，之后的提交就方便多了。
